Elasticsearch
Qu'est donc ?
  • Moteur d'indexation de données et de recherche basé sur Apache Lucène©
  • Architecture distribuée
  • Approche documentaire JSON, sans schéma
  • Interface REST
  • Modulaire : sites, rivières, plugins divers...
Terminologie
Un peu de vocabulaire
  • Shard : Plus petite unité de traitement. Correspond à une instance de Lucène.
  • Index : Ensemble de shards. C'est là que l'on insère les données.
  • Noeud : Ensemble d'index.
  • Cluster : Ensemble de noeuds.
Architecture
Généralités
  • Architecture distribuée (horizontale) : répartition de charge + tolérance aux pannes
  • Les noeuds peuvent être configurés pour ne pas porter de données et ainsi servir à la répartition de la charge.
  • Plusieurs types de noeuds :
    • Maître : 1 seul par cluster. Tous sont, par défaut, éligibles à ce poste mais un seul est choisi pour assuré ce rôle.
    • Réplicat : Noeud qui contient les shards d'un noeud configuré pour être répliqué.

Données
  • Approche documentaire sans schéma -> pas de format, tout type de données
  • Comment alimente-t-on un index :
    • En tirant : Rivières. Probablement dépréciées dans un futur proche
    • En poussant : programmatiquement ou avec Logstash
